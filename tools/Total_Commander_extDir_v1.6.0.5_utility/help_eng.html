Chapter 1: Introduction to extDir

  • About the utility

This small freeware utility extends your Total Commander's directory creating capabilities - after installing it, you can create a bunch of (even multi-leveled) directories with a single click, with numerous possibilities...

In other words, it can be used to replace Total Commander's F7 "Create Directory" dialog with an enhanced one, with much more available features.

  • Main Features
    • Built-in installer & uninstaller for easier usage
    • Normal, Extended & Batch (Decimal / Roman / ABC) directory creation mode
    • Full integration to Total Commander (F7, Shift + F7, or any key combination)
    • Multi-level directory creation (create directories with more than one subdirs)
    • Templates: create directory using user templates (e.g. "New Dir (%D-%N-%y)")
    • Replacements: use environment variables (like %USERNAME%) in the name dirname
    • Basic & special variables: use vars like %s (current second), %h (current hour), %x (unix time), {pc} (current PC name) etc
    • Multi-language support: English, Hungarian, Russian, Dutch, Spanish, German, Danish, Chinese, Ukrainian & Korean language support (built-in)
    • Intelligent selection: press Ctrl + Tab to select the next part of the dirname
      (works in Normal mode & for the manual path editor)
    • VirtualPanel (wfx) Total Commander plugin support
    • 6 built-in window color themes & the ability to create themes
    • Edit the current path manually (by double-clicking on the path)
    • Configuration & Template Manager dialogs
    • Enter directory after creation (optional)
    • Close extDir after directory creation (optional)
    • Undo function (no more accidentally lost dirnames)
    • Control the utility by hotkeys
    • Live Replacements (optional)
    • Drag & Drop support
    • ... and much more!
  • System Requirements

    Total Commander: 7.00+
    (heavily tested under Total Commander 7.56a, 7.57a & 8.0ÃŸ1-8.01)

    Windows: XP / Vista / 7 / 8
    (heavily tested under Windows 7 SP1 & XP SP3) [may also work under Windows 2000]

Chapter 2: Installation Guide

  • Important notice
    • You have to run the utility's executable file (tc extDir.exe) out of Total Commander's main directory, otherwise the installer won't show up.

      Example:
  • Other prerequisites
    • Supported operating system (check the list above)
    • Supported Total Commander version (7.00+)
    • Read & write access for Total Commander's base folder (by default: c:\totalcmd)
    • Read & write access for wincmd.ini file (or the according configuration file)
      [for installation]
    • Run the utility's installer with administrator rights if you want to be sure that every needed access is available
  • Installation guide (automatic)

    After you start the utility's executable file (out of Total Commander's directory), the installer automatically opens. Generally theres no need to configure anything, since it automatically selects the best settings depending on your Total Commander's language, however there are 3 things which you can manually modify:

    1. You can choose the language of extDir by clicking on the blinking blue arrow (on the top) then clicking on the name of the preferred language. Click on the arrow again to return to the general configuration page of the installer.
    2. You can modify the name of the installation subfolder inside Total Commander's directory (just click on the blue text under the disabled "Install to Total Commander's main directory" checkbox). You can use any executable & any subdirectory name - it's totally your choice. The only thing which can't be modified is to install extDir inside Total Commander's directory.
    3. You can set the activation hotkey: it's the hotkey, with which you can start the utility from Total Commander. By choosing Shift + F7 option a keyboard icon appears: you can set any activation hotkey by clicking on this icon.

You can start the installation by clicking on the "Install" button.

The utility will keep notifying you about the current progress of the installation until the end of the process.

Important: at the start of the installation the utility automatically tries to close the currently opened instance(s) of Total Commander. If it fails, the installation may won't be able to start until you manually closes all the opened instances of TC.

Good to know: for security reasons the utility automatically creates a backup of your "wincmd.ini" file, which will be created in the same directory, with "wincmd.old" name.

  • Installation guide (manual)

    Unzip the previously downloaded "tc extDir.exe" to the Total Commander's folder (you can put it in a subdir inside TC's folder, if you'd like to), then open Total Commander's Configuration/Options window, and add a new user command - which will be activated on F7 or Shift + F7 hotkey - with the name "em_extDir", and point it to "{the_path_to_your_totalcommander}\{extdir_subdirectory}\tc extDir.exe".

Chapter 3: Basic Functions (Normal mode)

  • Create a single directory

After starting extDir (pressing the activation hotkey in Total Commander) the utility's Normal mode is what you see (if you haven't changed it in the configuration file manually).

The utility automatically detects the currently selected file / directory in Total Commander (if nothing is selected, it detects the file under cursor), and sets the directory edit control's text to this string. You can type in (or paste) any directory name into this control. You can create the directory by pressing Enter, or by clicking on the OK button.

Note: You will see "Enter" button instead of "OK" in case the directory already exists in the current path - this way when you press the button or click on it, the utility will make Total Commander enter to this directory instead of trying to create it.

  • Create a single directory with subdirectories

You can also create directory / directories with one, or more than one subdirectories. Let's see an example:

Initial path: d:\Install\Programs\
Directory string: File managers\Total Commander\8.x
Result: d:\Install\Programs\File managers
  d:\Install\Programs\File managers\Total Commander
d:\Install\Programs\File managers\Total Commander\8.x

That's not a hard one though, and by the way Total Commander is also capable of doing the same thing using the internal F7 "New Folder" dialog.

Let's check the same with more than one subdirectories per subdirs:

Initial path: d:\Install\
Directory string: Programs\Audio players* Winamp\2.x\Installer* Winamp\5.x\Installer* AIMP* foobar2000
Result: d:\Install\Programs\Audio players
  d:\Install\Programs\Audio players\Winamp
d:\Install\Programs\Audio players\Winamp\2.x
d:\Install\Programs\Audio players\Winamp\2.x\Installer
d:\Install\Programs\Audio players\Winamp\5.x
d:\Install\Programs\Audio players\Winamp\5.x\Installer
d:\Install\Programs\Audio players\AIMP
d:\Install\Programs\Audio players\foobar2000

Maybe the directory string looks a bit complicated for the first look, but it isn't at all: after you copied the string to the directory edit combobox (in Normal mode), just click on the Extended button in the utility (or press F7 inside extDir), and you will understand how does it works.

The main pont is: every * means a unique subdirectory.

Important: Please note that this is just a small example to show you how to create multiple subdirectories in Normal mode - it is highly recommended to use the Extended mode of the utility for such situations, where you can give the data in list format.

  • Create multiple directories with separator

Okay, let's check another example with | separators (each | separator is equal to an enter character in Extended mode):

Initial path: c:\
Directory string: Backup\2k11\Test|Movies|Software\Freeware
Result: d:\Install\Programs\File managers
  d:\Install\Programs\File managers\Total Commander
d:\Install\Programs\File managers\Total Commander\8.x

Important: Please note that at the time of writing this documentation extDir doesn't supports creating more than one subdirectories (*) along with separators in Normal mode. You can use Extended mode in such cases.

  • Templates and Replacements (variables)

One of the main features of extDir is that you can use any variable in the name of your directory / directories. You can also create templates containing your most-used variables - this way you can access them by a single click / keypress.

Currently there are 4 categories of variables:

  • Date
  • System
  • Applications
  • Env. variables

You can always check the list of the variables by clicking on the + button on the top-right corner of extDir's window, or by pressing Ctrl + H.

What is a variable (replacement) good for?

It can be used for many things. For example you're often working on different computers, and you need to create folders which contains the name of the current PC many times a day.

By default you can only reach it by typing in (or pasting from clipboard) the name of the current PC, which is not a quick job (especially for names like "P4-S1_Z9X97BNAS").

In extDir you can simply press Ctrl + M (or click on Options / Manage templates), then type in the following, and press Apply:

Template: {pc} - {un}'s folder
Name: OS and username (or anything you prefer)
Result: BLUESTAR-NB\Bluestar's folder

Then you can access the newly created template - which contains the most used variables - by clicking on the template combobox (bottom of extDir's main window), or by pressing the Ctrl + T hotkey.

When the list is in dropdown state (or focused), you can simply press the keyboard button which is equal to the first alphabet of your template's name: in the example above it is O. This case you don't even need to click on a template, it will be activated by a single keypress.

  • Modifiers (space2Dot and its invert, changeCase, etc.)

You can currently use 3 type of modifiers - as it's name says, each one of them will modify the selected part of the directory string (in case there's no selection, then the whole string).

The 3 modifiers are:

  • Space to Dot (Ctrl + 1)
  • Dot to Space (Ctrl + 2)
  • Invert Case (Ctrl + 3)
  1. Space to Dot

Replaces the " " (space) characters inside the selected text to "." (dot).

Please note: IntelliModifiers feature is activated by default - that means not all of the characters will be replaced to its replacement, only which is between two other (non-space) characters. You can click here to check how to deactivate this feature.

Example 1: It is the name of a directory —› It.is.the.name.of.a.directory
Example 2: It is          another directory —› It.is         another.directory

  1. Dot to Space

Replaces the "." (dot) characters inside the selected text to
" " (space).

The same note applices to Dot to Space modifer (IntelliModifiers active).

  1. Invert Case

Inverts the currently selected text's case.

Example: DIRNAME with LeTtErS —› dirname WITH lEtTeRs

The same note applices to Dot to Space modifer (IntelliModifiers active).

  • Setting the directory's attributes

You can also set the directory's attributes in the utility. Just click on the small grey triangle on the top-left section - next to the "New folder (directory)" text -, or press Ctrl + Q for the attributes popup.

If you prefer to manually type in the needed attributes, you can press
Ctrl + Shift + Q, then type in one (or all) of the r a h s letters.

Note: setting the directory attributes will fail in case the current path is unavailable, and the utility has to create it. This behavior may change in future versions.

  • Additional options

Enable replacements
This option is automatically activated in case the directory string contains one or more variables / replacements. You can untick the checkbox if you don't want to activate replacements for the current session (this way it won't be automatically rechecked for the session if you type in variables).

Enter directory after creation
In case its checked, Total Commander will automatically enter to the directory after creation. The option is disabled in case you're creating more than one directories in Extended mode.

Close the window after creation
If this option is checked, extDir will be closed after successful directory creation. You can uncheck it if you'd like to create more directories at once.
